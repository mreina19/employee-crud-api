A Python REST API that lets you manage employees through a RESTful interface, with JWT authentication and role-based access control.
Built with FastAPI, SQLAlchemy, and PostgreSQL.

## Changelog

See [CHANGELOG.md](./CHANGELOG.md) for a history of changes to this project.

## Project Structure

| File | Description |
|------|-------------|
| `main.py` | Registers all HTTP routes and handles request/response logic |
| `models.py` | Defines the `User` entity that maps to the database table |
| `schemas.py` | Pydantic schemas that shape what goes in and out of the API |
| `database.py` | Database engine, session factory, and `get_db` dependency |
| `auth.py` | JWT creation, verification, and password hashing |
| `dependencies.py` | FastAPI dependencies for authentication, user validation, and admin enforcement |
| `screenshots/` | Screenshots of the API in action and the database |

## Endpoints

### Public
| Method | Route | Description |
|--------|-------|-------------|
| GET | `/` | Health check / welcome message |
| POST | `/token` | Logs in a user and returns a JWT access token |

### Authenticated
| Method | Route | Description |
|--------|-------|-------------|
| GET | `/profile` | Returns the currently logged in user |
| GET | `/verify-token` | Confirms the token is valid and returns basic user info |

### Admin only
| Method | Route | Description |
|--------|-------|-------------|
| POST | `/register/` | Creates a new user |
| GET | `/users/` | Returns all users |
| GET | `/users/{user_id}` | Returns a user by ID |
| PUT | `/users/{user_id}` | Updates an existing user |
| DELETE | `/users/{user_id}` | Deletes a user |

## Requirements

- Python 3.10+
- PostgreSQL

### Python Packages

| Package | Purpose |
|---------|---------|
| `fastapi` | Web framework |
| `uvicorn` | ASGI server to run the app |
| `sqlalchemy` | ORM and database toolkit |
| `psycopg2-binary` | PostgreSQL driver |
| `pydantic` | Data validation and serialization |
| `pydantic[email]` | Enables `EmailStr` email validation |
| `python-dotenv` | Loads environment variables from `.env` |
| `passlib[bcrypt]` | Password hashing using bcrypt |
| `PyJWT` | JWT creation and verification |
| `python-multipart` | Required for OAuth2 form data parsing |

Install all dependencies:

```bash
pip install fastapi uvicorn sqlalchemy psycopg2-binary pydantic pydantic[email] python-dotenv passlib[bcrypt] PyJWT python-multipart
```

### Running the Project

1. Clone the repository
2. Create a `.env` file in the project root with the following variables:
```env
DATABASE_URL=postgresql://user:password@localhost:5432/your_database
SECRET_KEY=your_secret_key_here     # see step 3
ALGORITHM=HS256                     # recommended
TOKEN_EXPIRES=30                    # recommended
```
3. Generate a secure secret key and replace the `SECRET_KEY` value in `.env`:
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```
4. Run the application once to create the database tables, then stop it:
```bash
uvicorn main:app
```
5. Insert the first admin user directly in the database. To generate the hashed password, create a new Python file with the following content and run it:
```python
from passlib.context import CryptContext
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")
print(pwd_context.hash("your_password"))
```
Then copy the output and use it in the INSERT statement:
```sql
INSERT INTO users (first_name, last_name, email, role, hashed_password, is_active)
VALUES ('Admin', 'User', 'admin@company.com', 'admin', 'your_hashed_password', true);
```
6. Run the application:
```bash
uvicorn main:app --reload
```
7. Open the interactive API docs at:
```
http://localhost:8000/docs
```

## Notes

- The `id` is always generated by the database. It is never sent in POST or PUT request bodies.
- `POST /register/` returns `201 Created` on success.
- `DELETE` returns `204 No Content` on success.
- Duplicate emails are not allowed. `Email` must be unique (case insensitive).
- Passwords are hashed with bcrypt before being stored. Plain text passwords are never saved.
- `role` must be one of: `admin`, `manager`, `developer`, `designer`, `analyst`, `hr`.
- `first_name` and `last_name` are limited to 50 characters, `email` to 70 characters, `password` to 72 characters (bcrypt limit).
- Users can be deactivated (`is_active = false`) without being deleted, either through `POST /register/`, `PUT /users/{user_id}`, or directly in the database.
- All user management routes require admin privileges. Non-admin users will receive a `403 Forbidden` response.
- The first admin user must be set manually in the database. After that, admins can create and manage other users through the API.
- To access protected routes in the API docs, click the **Authorize** button in the top right corner of `/docs`, enter your credentials, and routes will work according to your role for the duration of the token.
- Log entries only appear when the request is authenticated. Unauthenticated requests are rejected by FastAPI before reaching the route function, so no application-level log is generated. Only `POST /token` always logs regardless of authentication.
- All error responses follow standard HTTP status codes (`401` unauthorized, `403` forbidden, `404` not found, `409` conflict, `500` internal error).
